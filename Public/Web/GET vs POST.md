# GET

> 정보를 가져와서 조회하기 위함

- URL에 변수를 포함시켜 요청한다.
  - google.com/search**?input=SSAFY**와 같이 ?뒤에 key=value의 형태로 붙인다.
- 데이터를 Header에 포함하여 전송한다.
- 따라서, 데이터가 노출되기 쉬워 보안에 취약하다.
- URL에 붙여서 쓰기때문에 길이제한이 있다.

- Body는 보통 비어있다.



# POST

> 데이터를 서버에 전송하여 요청한 후 추가 또는 수정하기 위함

- 데이터를 Body에 포함시켜서 요청
- 길이제한이 없음
- 보안이 GET에 비해 상대적으로 우수
